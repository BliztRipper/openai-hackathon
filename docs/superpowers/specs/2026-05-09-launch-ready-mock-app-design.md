# Second Brain Launch-Ready Mock App Design

## Goal
Revamp the current Vite demo into a ready-to-launch workspace-data product surface that immediately communicates the PRD thesis: Second Brain preserves cognitive agency in a post-AGI world by adding gentle cognitive friction, anchoring reality, and summarizing support trends for caregiver/clinician review.

## Product shape
The app remains a local/static-first MVP with optional JSON/API persistence. It should feel like a polished health-adjacent SaaS/wellness product, not a raw hackathon wizard. The default flow is a guided product demo for judges, while every screen uses product language that could ship with mock data.

## Visual direction
Use DESIGN.md as source of truth: Porcelain/Mist Blue/Sage Green base, Graphite text, restrained Coral urgency, Deep Lake primary actions, soft glass surfaces, large readable type, calm nature/organic atmosphere, no emoji structural icons, and non-diagnostic language. UI must be elderly-first: large text, 44px+ targets, clear focus states, low typing burden, voice-first framing.

## Core surfaces
1. Launch hero with value proposition, live evidence cards, consent status, and a demo CTA.
2. Persona selector for Malee, Somchai, and Araya with persona-specific risks and scenario clues.
3. Consent/data boundary screen that explains synthetic sources, permission tiers, and no raw surveillance.
4. Companion loop showing hands-free/voice-first transcript, recall cue ladder, real-time signal emission, and API/mock insight seam.
5. Memory architecture page connecting episodic/semantic/procedural/working stores to dashboard evidence.
6. Care dashboard with cognitive-domain trends, necessary notices, validation-readiness mapping, capability-inflation examples, companion-boundary signals, and JSON review queue.

## Data requirements
Use mock data only. Cover PRD domains: memory, planning, communication/language, financial management, daily routine, orientation, social cue perception, AGI dependence, human contact, validation-readiness, autonomous-vs-assisted writing/finance gaps. Preserve current API/client contracts where possible.

## Safety and quality
Avoid diagnosis. Use uncertainty language. Show support recommendations and next review steps. Provide accessible charts with labels/summaries. Keep responsive layouts working at 375px, tablet, and desktop. Respect reduced motion.
